AI agent 하네스 관련 논문리뷰 10편

1. Agentic Harness Engineering 2026/04
하네스가 중요한걸 알지만 아직 하네스설계는 수작업이고 이를 자동화하는것은 heterogeneous한 공간(action space)들과 같은 어려움을 겪음

1. 수백만 토큰의 raw trace에서, 막 multi step에 skill.md 쓰고 tool call, subagent, MCP 이렇게 쓰면 "어떤 컴포넌트가 왜 실패했는지" 알기 어려움

2. 컴포넌트 간 강한 결합 → 편집이 불안정
기존 하네스 프레임워크는 prompt/tool/middleware가 얽혀있어서, 하나 건드리면 다른 게 망가짐. 편집의 action space 자체가 불명확

->

"이 문제는 에이전트 능력의 문제가 아니라 관찰가능성의 문제다."
evolution 에이젼트에게 구조화된 컨텍스트와 명확한 action space를 주면, 더 나은 하네스를 찾아낼 수 있다"

Method
NexAU에서 시도 - Hermes, 클로드 코드 열화판 오픈소스

7개 컴포넌트 타입, 각각의 파일로 분리:
System prompt (systemprompt.md)
Tool description (.tool.yaml)
Tool implementation (tools/*.py)
Middleware (middleware/*.py)
Skill (skills/*/SKILL.md)
Sub-agent configuration (sub_agents/*/agent.yaml)
Long-term memory (LongTermMEMORY.md)

실제 evolve 프롬프트는 "workspace/ 디렉토리만 당신의 작업 공간입니다. 나머지는 읽기 전용이거나 수정 불가입니다" 이런식
